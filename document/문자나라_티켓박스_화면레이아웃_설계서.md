## 1. 공통 레이아웃 시스템

### 1.1 뷰포트 & 안전영역

| 항목 | 태블릿 가로 (primary) | 키오스크 세로 (secondary) |
|---|---|---|
| 물리 해상도 | 1280 × 800 | 1080 × 1920 |
| CSS 뷰포트 | `100vw × 100vh` | `100vw × 100vh` |
| Safe Padding (X / Y) | `48px / 32px` | `72px / 96px` |
| 컨텐츠 최대폭 | 1184 (1280 - 48×2) | 936 (1080 - 72×2) |
| 컨텐츠 최대높이 | 736 (800 - 32×2) | 1728 (1920 - 96×2) |
| 기본 폰트 스케일 | 1.0× (root 16px) | 1.25× (root 20px) |

### 1.2 12-컬럼 그리드 (태블릿 가로)

```
<──────────────────── 1280 ────────────────────>
│ 48 │ col ── gutter 24 ── col ── ... ── col │ 48 │
        한 컬럼 폭 = (1184 - 24×11) / 12 ≈ 76.7px
```

| 속성 | 값 |
|---|---|
| Column count | 12 |
| Gutter | 24px |
| Outer padding X | 48px |
| Column width (계산값) | 76.7px |
| Baseline (수직 리듬) | 8px |
| 공용 헤더 높이 | 72px |
| 공용 푸터(힌트바) 높이 | 64px |
| **컨텐츠 본문 높이** | **800 - 72 - 64 = 664px** |

### 1.4 Z-Index 레이어 규약

| 레이어 | z-index | 용도 |
|---|---:|---|
| base | 0 | 페이지 본문 |
| ai-character | 10 | AI 캐릭터 스프라이트 (페이지 위 고정) |
| speech-bubble | 20 | 말풍선 / 자막 |
| overlay | 30 | 로딩 연출·소프트 마스크 |
| modal | 40 | 확인 다이얼로그 ("지안이 맞아?") |
| toast | 50 | 에러·안내 토스트 |

### 1.5 터치 타겟 규칙

- 최소 터치 영역: **120×120px** (버튼 실 렌더 크기는 이보다 작아도 되지만 hit-area는 120 이상)
- 카드 간격: 24px 이상
- 손가락 당 여유: 버튼끼리 24px 이상 이격
- 화면 가장자리 12px 이내엔 인터랙션 요소 **배치 금지** (오터치 방지)

## 2. 공통 셸 컴포넌트

### 2.1 `<AppShell>` — 모든 페이지의 래퍼

```
┌────────────────────────────────────────────────────────────┐
│ <TopBar/>                                              72px │  ← header
├────────────────────────────────────────────────────────────┤
│                                                             │
│                                                             │
│                  <Outlet/>   (664px)                        │
│                                                             │
│                                                             │
├────────────────────────────────────────────────────────────┤
│ <HintBar/> · 현재 단계 인디케이터                          64px │  ← footer
└────────────────────────────────────────────────────────────┘
```

컴포넌트 트리:

```tsx
<AppShell locale={locale} stage={stage}>
  <BackgroundLayer variant="paper" />   {/* position:fixed, z:0 */}
  <TopBar />                            {/* z:10, sticky */}
  <main className="flex-1 px-12 py-0">
    <AnimatePresence mode="wait">
      <Outlet key={stage} />            {/* page content */}
    </AnimatePresence>
  </main>
  <HintBar stage={stage} />
  <AdminSecretTap />                    {/* 우상단 48×48 투명 영역, 3회 탭 */}
</AppShell>
```

### 2.2 `<TopBar>` — 상단 고정 바

| 영역 | 위치 | 내용 | 크기 |
|---|---|---|---|
| 좌측 로고 | `x: 48, y: 16` | 박물관 심볼 + "문자 나라" 로고 | 40×40 + 텍스트 |
| 중앙 스테퍼 | `x: center, y: 24` | S0~S6 단계 인디케이터 (점 6개, 현재 단계 확대) | 280×24 |
| 우측 언어 토글 | `x: 1232-W, y: 16` | `<LanguageToggle>` (KR/EN/JP/CN) | 48×160, pill |

- `<AttractPage>`, `<GeneratingPage>` 에서는 TopBar `opacity:0.4` (몰입 강화)
- `<AdminDashboard>`에서는 TopBar 미노출, 자체 헤더 사용

### 2.3 `<HintBar>` — 하단 힌트 바

| 영역 | 용도 |
|---|---|
| 좌측 (160px) | 뒤로가기 버튼 (S1 이상에서만 노출, 24×24 아이콘 + "뒤로") |
| 중앙 (flex) | 상황별 힌트 (예: "아무거나 터치해봐!", "3개 골라줘") |
| 우측 (160px) | 보조 CTA ("직접 입력", "결정!" 등) |

### 2.4 `<AICharacter>` 배치 규칙

AI 캐릭터는 **페이지마다 배치가 달라짐**. 각 페이지 레이아웃 §3에서 명시.

공통 애니메이션 상태:
- `idle` (호흡 스케일 0.98→1.02, 2s ease-in-out loop)
- `talking` (입모양 + 살짝 상하 움직임, TTS 재생 중)
- `thinking` (머리 위 ... 물음표, 3초 이상 무응답 시)
- `happy` (점프 한 번, 캐릭터 생성 성공 시)
- `waving` (손 흔들기, S6)

## 3. 시나리오별 화면 레이아웃

### S0. Attract / Idle — `/`

#### (1) 목적
지나가는 관람객의 관심 유도. 3초 안에 "재미있겠다"를 만든다.

#### (2) ASCII 와이어프레임 (1280×800)

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                       │
│   ╭──────────────────────────────────────────────────────────────╮   │
│   │                                                               │   │
│   │              ✨ 문 자  나 라  티 켓 박 스 ✨                 │   │  ← h1 (64px)
│   │                                                               │   │
│   │        ┌───────────────────────────────────────────┐         │   │
│   │        │                                            │         │   │
│   │        │        (AI 캐릭터 루프 영상 480×480)       │         │   │
│   │        │              ← 손짓 애니메이션             │         │   │
│   │        │                                            │         │   │
│   │        └───────────────────────────────────────────┘         │   │
│   │                                                               │   │
│   │              ▶  화면을 터치해서 시작! 👆                     │   │  ← h3 pulse
│   │                                                               │   │
│   ╰──────────────────────────────────────────────────────────────╯   │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
   전체 터치 영역 (onTouchStart → startSession)
```

#### (3) 그리드 스펙

| 영역 | x / y | 폭 × 높이 | 정렬 | 비고 |
|---|---|---|---|---|
| 타이틀 | `center, y=120` | `800×88` | center | 반짝임 파티클 뒤에 배경 |
| 캐릭터 영상 | `center, y=230` | `480×480` | center | `<video loop muted autoPlay>` |
| 터치 힌트 | `center, y=720` | `auto×40` | center | 1.2s 펄스 애니메이션 |

**TopBar / HintBar 미노출** — 전체 캔버스 1280×800 몰입 모드.

#### (4) 컴포넌트 트리

```tsx
<AttractPage onEnter={startSession}>
  <FullBleed onPointerDown={handleTouch}>   {/* 전체 화면 터치 캐처 */}
    <ParticleBackdrop />                    {/* 파피루스 + 반짝이 */}
    <Title className="absolute top-[120px] inset-x-0 text-center">
      문자 나라 티켓박스
    </Title>
    <CharacterLoopVideo
      src="/assets/attract-loop.webm"
      className="absolute top-[230px] left-[400px] w-[480px] h-[480px]"
    />
    <TouchHint
      className="absolute bottom-[56px] inset-x-0 text-center animate-pulse-slow"
    >
      화면을 터치해서 시작!
    </TouchHint>
  </FullBleed>
</AttractPage>
```

#### (5) 상태 · 이벤트

| 이벤트 | 동작 |
|---|---|
| `onPointerDown` (어디든) | `sessionStore.startSession()` → `navigate('/intro')` |
| `visibilitychange: hidden` | 영상 일시정지 (전력 절감) |
| idle 60s+ | 영상 계속, 추가 액션 없음 |

#### (6) 애니메이션 스펙

| 요소 | 모션 |
|---|---|
| Title 글리터 | Framer Motion `initial={{y:-20, opacity:0}} animate={{y:0, opacity:1}} transition={{duration:0.8}}` 후 반복 없음 |
| TouchHint pulse | `animate={{scale:[1,1.05,1]}} transition={{repeat:Infinity, duration:1.2}}` |
| Character Video | `<video>` 무한 루프, `playbackRate=1` |
| 파티클 | Canvas (tsparticles), 20~30개, drift 0.3~0.6px/frame |

#### (7) 키오스크 세로(1080×1920) 적응

- 타이틀 `y=220`, 폰트 `96px`
- 캐릭터 영상 720×720, `x=180, y=440`
- 터치 힌트 `y=1720`, 폰트 `56px`

#### (5) 상태 · 이벤트

| 상태 | 조건 | 동작 |
|---|---|---|
| `speaking` | 페이지 진입 시 | TTS 재생 + 자막 타이핑 |
| `ctaVisible` | TTS `onEnded` or 타이핑 종료 | CTA fade-in (duration 0.4s) |
| 언어 변경 | `<LanguageToggle>` | `setLocale()` → TTS 재요청 → 말풍선 재출력 |

| 이벤트 | 동작 |
|---|---|
| CTA 클릭 | `navigate('/name')` |
| TTS 자동완료 + 3초 후 CTA 미클릭 | AI `thinking` → "준비됐어?" 추가 대사 |
| HintBar 뒤로 | `navigate('/')` + `resetSession()` |

#### (6) 애니메이션 스펙

| 요소 | 모션 |
|---|---|
| 페이지 진입 | `AnimatePresence` slide from right, 0.4s ease-out |

| 캐릭터 talking | 살짝 상하 ±4px, 0.3s loop |

#### (7) 키오스크 세로 적응

- 2-col 그리드 → 1-col **세로 배치**: 캐릭터 위(560×560), 말풍선 아래(880×400)
- CTA는 `y=1600`, `680×120`

### S3. 문자 선택 (Script Selection) — `/scripts`

> **v1.1 업데이트** — 인터랙션 모델 변경: "3가지 문자 일괄 선택" → **"이름 각 음절마다 한 문자씩 골라 조립"**
> 예: **"나민원"** → `나`에 娜 / `민`에 ミン / `원`에 𓁹 → 완성 표기 "娜ミン𓁹"

#### (1) 목적
사용자 이름을 **음절 단위로 분해**해, 각 음절마다 같은 소리의 다양한 문자(라틴·한자·히라가나·가타카나·태국·그리스·이집트 상형·쐐기)를 **주변에 떠다니는 카드**로 보여주고, 아이가 마음에 드는 문자를 터치해 **음절 아래 슬롯에 떨어뜨리는** 방식으로 "세계 문자가 섞인 나만의 이름"을 완성한다.

이 방식은:
- **원래 이름의 시각적 골격 유지** — 한글 음절이 그대로 보존되므로 "내 이름"의 정체성이 유지됨
- **탐색의 재미 극대화** — 같은 "나" 소리가 문자별로 얼마나 다르게 생겼는지 **직접 비교 가능**
- **난이도 3단계 자연스럽게 수용** — 난이도 = 후보 문자 풀의 구성 (하: 영/일, 중: +한자/그리스, 상: +상형/쐐기)

interface SyllableChoice {

  text: string;              // "娜"

interface ScriptSelectSlice {
  syllables: string[];                                  // ['나','민','원']

  candidates: Record<number, Candidate[]>;              // index별 후보 풀

## 4. 공통 모션 & 전환 스펙

### 4.1 페이지 전환 (React Router + Framer Motion)

```tsx
// router.tsx
<AnimatePresence mode="wait">
  <motion.div
    key={location.pathname}
    initial={{ opacity: 0, x: 40 }}
    animate={{ opacity: 1, x: 0 }}
    exit={{ opacity: 0, x: -40 }}
    transition={{ duration: 0.35, ease: [0.2, 0.8, 0.2, 1] }}
  >
    <Outlet />
  </motion.div>
</AnimatePresence>
```

### 4.2 공용 Easing / Timing

| 용도 | duration | easing |
|---|---|---|
| 페이지 전환 | 350ms | `cubic-bezier(0.2, 0.8, 0.2, 1)` |
| 버튼 피드백 | 120ms | `ease-out` |
| 카드 hover / press | 180ms | `ease-out` |
| 모달 open | 240ms | spring (stiffness 240, damping 22) |
| 강조 pulse | 1200ms | ease-in-out (loop) |

### 4.3 모션 축약 토큰 (헬퍼 상수)

```ts
// src/utils/motion.ts
export const motion = {
  pageSlide: { initial:{opacity:0,x:40}, animate:{opacity:1,x:0},
               exit:{opacity:0,x:-40}, transition:{duration:0.35}},
  fadeUp:    { initial:{opacity:0,y:16}, animate:{opacity:1,y:0}},
  pop:       { whileTap:{scale:0.96}, transition:{duration:0.12}},
  ticketOut: { initial:{y:400,rotate:-2}, animate:{y:0,rotate:0},
               transition:{type:'spring',stiffness:80,damping:14}},
};
```
